# Closing Checklist — GigaSaudi.com Direct Sale

**Asset:** GigaSaudi.com  
**Managed through:** Sohadot (agent@sohadot.com)  
**Classification:** Internal · Use for each qualified closing

---

## Pre-Closing: Qualification and Agreement

- [ ] Buyer identity confirmed (legal name, organisation, email)
- [ ] Buyer email on record matches verified organisational identity
- [ ] Buyer’s intended use reviewed and assessed as qualified
- [ ] Sale scope agreed in writing (Option A, B, or C)
- [ ] Purchase price agreed in writing
- [ ] Escrow provider agreed by both parties
- [ ] Escrow fee responsibility agreed (buyer / seller / split)
- [ ] Transfer method agreed (registrar push or registrar transfer)
- [ ] Inspection period length agreed
- [ ] Transaction terms (ACQUISITION_TERMS_DRAFT.md or equivalent) acknowledged by both parties

---

## Escrow Setup

- [ ] Escrow transaction opened at agreed provider
- [ ] Transaction details entered correctly: asset name, sale scope, purchase price, transfer method
- [ ] Both parties added to escrow transaction
- [ ] Buyer instructed to fund escrow
- [ ] Buyer funds confirmed received and secured in escrow
- [ ] Seller notified of escrow confirmation

---

## Domain Preparation

- [ ] Domain confirmed active and in good standing at registrar
- [ ] No pending disputes, legal claims, or third-party holds confirmed
- [ ] Domain privacy / WHOIS lock status reviewed
- [ ] If registrar push: confirm buyer has account at current registrar; push prepared
- [ ] If registrar transfer: confirm no 60-day lock is active; domain unlocked; EPP auth code obtained and ready

---

## Transfer Execution

- [ ] Transfer initiated (push or auth code provided to buyer / escrow as agreed)
- [ ] Transfer status monitored
- [ ] Transfer confirmed complete at registrar level
- [ ] Buyer notified to confirm receipt

---

## Buyer Confirmation and Funds Release

- [ ] Buyer confirms receipt and control of domain in writing (email to agent@sohadot.com and to escrow)
- [ ] Buyer confirmation includes: domain name, buyer registrar account, date of confirmed control
- [ ] Inspection period elapsed with no dispute raised (if applicable)
- [ ] Escrow instructed to release funds to seller
- [ ] Seller payout confirmed received

---

## Post-Closing

- [ ] Transaction records archived (inquiry, terms, escrow confirmation, transfer confirmation, buyer confirmation, payout confirmation)
- [ ] Website and/or repository status updated as agreed with buyer (transfer of GitHub repo if Option B/C)
- [ ] Any agreed transition support initiated (Option C only)
- [ ] Availability page updated or taken offline as appropriate post-closing
- [ ] Internal acquisition documents archived, not deleted
